Competitive Advantages Built Into Our Design

Three features neither competitor has:
1. Real-time tool call timeline — every read_file, edit_file, bash call shown live with expandable I/O. Claude Code hides this; you only see results.
2. Skill usage visibility cards — when the agent loads a skill, you see which one, for how long, and what it output. This is our "killer feature" for opencode users.
3. CodeLens inline actions — right-click a function → "Generate tests" → diff appears inline. Neither competitor offers this directly in the editor gutter.

Claude Code Features We Should Match (Add to Plan)

Based on this analysis, I recommend adding these to our design:
• @-mention system — typing @ in chat auto-completes file paths with line ranges
• Plan mode — read-only agent mode, plan opens as editable markdown for inline feedback
• Permission mode selector — normal/plan/auto-accept toggle in the chat UI
• Session history — searchable list of past conversations, resume with full context
• URI handler — vscode://opencode-harness/open?prompt=... for scripting
• Keyboard shortcut for file reference insertion — Alt+K inserts @filename#L-range
