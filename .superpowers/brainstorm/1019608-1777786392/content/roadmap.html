Project Layout

Directory Structure
opencode-harness/
├── .vscode/
│   ├── launch.json          # Debug config (F5 → Extension Dev Host)
│   └── tasks.json           # Build tasks
├── src/
│   ├── extension.ts          # Entry point: activate/deactivate
│   ├── session/
│   │   └── SessionManager.ts # Server lifecycle, SDK wrapper, SSE events
│   ├── context/
│   │   └── ContextEngine.ts  # Context gathering worker
│   ├── chat/
│   │   ├── ChatProvider.ts   # Webview provider (extension host side)
│   │   └── webview/          # Webview UI (HTML/JS/CSS or Svelte)
│   ├── checkpoint/
│   │   └── CheckpointManager.ts  # Git worktree snapshots
│   ├── diff/
│   │   └── DiffApplier.ts    # Diff generation + workspace.applyEdit
│   ├── inline/
│   │   └── InlineActionProvider.ts  # CodeLens + context menus
│   ├── skills/
│   │   └── SkillManager.ts   # Skill tree view + enable/disable
│   ├── monitor/
│   │   └── ContextMonitor.ts # Status bar ring + token estimation
│   ├── terminal/
│   │   └── TerminalBridge.ts # Output channel + terminal capture
│   └── utils/
│       ├── portFinder.ts     # Dynamic port allocation
│       └── tokenCounter.ts   # tiktoken-based estimation
├── package.json              # Extension manifest + contributes
├── tsconfig.json
└── README.md

Non-Functional Requirements Summary

Category Requirement Target
Startup Extension must not increase VS Code startup time <50ms
Responsiveness Chat input stays responsive during AI generation No UI thread blocking >16ms
Context estimation Token count for context package <10ms computation
Reliability Graceful recovery if opencode server crashes Auto-restart, preserve history
Reliability All file operations are transactional diff → review → apply (never direct write)
Platform Arch Linux + Fedora/RHEL first-class support Tested on both before v1.0
