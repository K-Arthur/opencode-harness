Interaction Sequence: User Sends a Chat Message

End-to-end flow from keystroke to inline diff acceptance

Sequence Diagram
Phase 1: Context Gathering (async, non-blocking)
1 User types in Chat → ChatProvider Captures message text, flags as pending
2 ChatProvider → ContextEngine Request context package
3 ContextEngine → VS Code API Get open tabs, diagnostics, git status, workspace tree (worker thread: <10ms for basic, <50ms with AST)
4 ContextEngine → ChatProvider Returns structured ContextPackage
Phase 2: Checkpoint + Send (transactional)
5 ChatProvider → CheckpointManager Take git worktree snapshot Before any file writes occur
6 ChatProvider → SessionManager client.session.prompt({ parts: [...], context })
7 SessionManager → openCode Server POST /session/:id/message (blocking response)
Phase 3: Real-time Streaming (SSE events)
8 SessionManager → ChatProvider Emits: event.thinking, event.tool_call, event.skill_load SSE events from /event stream
9 ChatProvider → Webview UI Renders tool cards, skill cards, streaming text in real-time
10 ChatProvider → ContextMonitor Update context usage ring (estimates from message length)
Phase 4: Response + Code Application
11 openCode Server → SessionManager Response: AssistantMessage with parts[]
12 SessionManager → DiffApplier Parse code blocks, generate unified diff vs current file
13 DiffApplier → ChatProvider Render diff view with Accept / Reject / Edit buttons
14 User clicks Accept → DiffApplier workspace.applyEdit() — native VS Code undo stack

Key Design Decisions

Worker Thread for Context

ContextEngine runs in a Node.js worker thread. Gathering open tabs + diagnostics + git status never blocks the UI thread. Guaranteed <10ms for basic mode, with AST parsing as an optional premium mode.

SSE Events, Not Polling

Instead of polling for status, we subscribe to opencode's /event SSE stream. Every tool call, skill activation, and thinking step is pushed to us in real-time — exactly what we need for the agent visibility feature.

Git Worktrees for Rollback

Before any AI file write, CheckpointManager creates a lightweight git worktree snapshot. Rollback is instant — just checkout the previous ref. No file copying, no diff reversal, pure git.

Diff Before Apply

AI-suggested code is never written directly. It's compared against the current file state to produce a unified diff. User reviews and accepts; we apply via workspace.applyEdit() — fully undoable via Ctrl+Z.
