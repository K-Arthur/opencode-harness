## Issues Fixed (Severity Order)

### P0 — Critical

#### C-019: Duplicate `newTabBtn` click listener creates two tabs per click
- **File:** `src/chat/webview/main.ts` (line 745), `src/chat/webview/tabs.ts` (line 10)
- **Root cause:** `els.newTabBtn` received two `click` listeners:
  1. `tabs.ts::createTabBar()` — `callbacks.onNew()` → `createNewTab()`
  2. `main.ts::setupButtons()` — direct `createNewTab()` call
- **Impact:** Clicking the "+" button instantly created two session tabs (and two extension host sessions)
- **Fix:** Removed the listener from `setupButtons` in `main.ts` with a comment explaining the single-source-of-truth policy
- **Test:** `main.test.ts` — asserts `newTabBtn` has at most one listener

### P1 — High

#### C-016: Multiple sessions open on extension startup
- **File:** `src/chat/webview/main.ts` (line 983)
- **Root cause:** `init_state` handler checked for existing tabs using `querySelector("vscode-tab-panel")`. The actual DOM uses `.tab-panel` divs. This caused the check to always fail, potentially creating duplicate welcome tabs on re-init.
- **Impact:** While the current working tree already intended to show only the welcome screen on startup, the wrong selector was a latent bug that could reintroduce multiple tabs if the DOM structure changed.
- **Fix:** Changed selector to `querySelector(".tab-panel")`
- **Test:** `main.test.ts` — asserts `.tab-panel` is used and `vscode-tab-panel` is absent

#### C-017: Welcome screen not shown after closing last session
- **File:** `src/chat/webview/main.ts` (lines 296-337)
- **Root cause:** `closeTab()` had a guard `if (sessions.length <= 1) { createNewTab("Default") }` that created a new tab **before** closing. `createNewTab` uses `isWelcome = false` by default, so the new tab had no welcome content.
- **Impact:** After closing all sessions, a blank "Default" tab remained instead of the welcome screen with suggestion cards.
- **Fix:** Removed the premature guard. Moved session-creation to AFTER the close completes, calling `createInitialTab("Default")` (which passes `isWelcome = true`) only when `stateManager.getAllSessions().length === 0`.
- **Test:** `main.test.ts` — asserts `createInitialTab` is called and `createNewTab("Default")` without `isWelcome` is absent

#### C-018: Active session incorrectly shown in Recent list
- **File:** `src/chat/webview/main.ts` (lines 139-158)
- **Root cause:** `renderRecentSessionsList()` filtered for `s.messages.length > 0` but did not exclude the currently active session. The active session is "current," not "recent."
- **Impact:** The RECENT section showed the open session alongside truly recent sessions, causing confusion.
- **Fix:** Added `s.id !== activeId` to the filter predicate
- **Test:** `main.test.ts` — asserts active session exclusion logic exists
